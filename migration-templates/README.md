# EROS Database Migration Templates

These CSV templates show you what data you need to collect to populate your EROS ticketing system.

## File Order (Import in this sequence)

1. **01_users.csv** - All staff and creators need user accounts first
2. **02_creators.csv** - Creator profiles (linked to users with CREATOR role)
3. **03_user_creator_assignments.csv** - Which staff work with which creators
4. **04_creator_availability.csv** - When creators are available (optional)
5. **05_tickets_optional.csv** - Existing tickets to migrate (usually start fresh)

---

## 01_users.csv - User Accounts

Everyone who logs into the system needs a user account.

| Column | Required | Description | Example Values |
|--------|----------|-------------|----------------|
| email | Yes | Login email (unique) | john@agency.com |
| password | Yes | Temporary password (will be hashed) | TempPass123! |
| name | Yes | Display name | John Smith |
| role | Yes | Access level | SUPER_ADMIN, MANAGER, SCHEDULER, CHATTER, CREATOR |
| is_active | Yes | Account enabled? | true, false |

**Roles Explained:**
- `SUPER_ADMIN` - Full access, can manage everything
- `MANAGER` - Can see all tickets, reassign, generate reports
- `SCHEDULER` - Manages content schedules and creator calendars
- `CHATTER` - Creates tickets for assigned creators only
- `CREATOR` - Responds to their own tickets only

---

## 02_creators.csv - Creator Profiles

Only for users with role=CREATOR. Links to their user account.

| Column | Required | Description | Example Values |
|--------|----------|-------------|----------------|
| user_email | Yes | Must match a CREATOR user | creator@example.com |
| stage_name | Yes | Public/performer name | StarPerformer |
| platforms | Yes | Comma-separated list | onlyfans,fansly,chaturbate |
| timezone | Yes | IANA timezone | America/New_York, America/Los_Angeles |
| is_active | Yes | Taking new work? | true, false |

**Common Timezones:**
- America/New_York (Eastern)
- America/Chicago (Central)
- America/Denver (Mountain)
- America/Los_Angeles (Pacific)

---

## 03_user_creator_assignments.csv - Who Works With Whom

Links staff members to the creators they manage/work with.

| Column | Required | Description | Example Values |
|--------|----------|-------------|----------------|
| user_email | Yes | Staff member email | chatter@agency.com |
| creator_stage_name | Yes | Creator's stage name | StarPerformer |
| is_primary | Yes | Primary contact? | true, false |

**Notes:**
- A chatter can only create tickets for their assigned creators
- Multiple staff can be assigned to one creator
- Mark one as `is_primary=true` for main contact

---

## 04_creator_availability.csv - Scheduling (Optional)

When creators are available for video calls, streams, etc.

| Column | Required | Description | Example Values |
|--------|----------|-------------|----------------|
| creator_stage_name | Yes | Creator's stage name | StarPerformer |
| day_of_week | For recurring | 0=Sun, 1=Mon...6=Sat | 1 |
| start_time | Yes | Available from | 09:00, 18:00 |
| end_time | Yes | Available until | 17:00, 23:00 |
| is_recurring | Yes | Weekly pattern? | true, false |
| specific_date | For one-off | Specific date | 2025-02-14 |
| is_available | Yes | Available or blocked? | true, false |
| reason | For blocks | Why unavailable | Vacation, Personal |

**Recurring Example:** Available every Monday 9am-5pm
```
StarPerformer,1,09:00,17:00,true,,true,
```

**Block Example:** Blocked on Valentine's Day
```
StarPerformer,,,,false,2025-02-14,false,Valentine's Day
```

## 05_tickets_optional.csv - Existing Tickets

Usually you'll start fresh, but if you have existing tickets to migrate:

| Column | Required | Description | Example Values |
|--------|----------|-------------|----------------|
| ticket_number | Yes | Unique ID | CVR-2025-00001 |
| type | Yes | Ticket type | CUSTOM_VIDEO, VIDEO_CALL, CONTENT_REQUEST, GENERAL_INQUIRY, URGENT_ALERT |
| status | Yes | Current state | DRAFT, SUBMITTED, PENDING_REVIEW, ACCEPTED, IN_PROGRESS, COMPLETED, REJECTED, CANCELLED |
| priority | Yes | Urgency | LOW, MEDIUM, HIGH, URGENT |
| title | Yes | Short summary | Birthday Video for John |
| description | No | Full details | Fan wants a 5-min message... |
| creator_stage_name | Yes | For which creator | StarPerformer |
| created_by_email | Yes | Who submitted | chatter@agency.com |
| assigned_to_email | No | Who's handling | chatter@agency.com |
| deadline | No | Due date | 2025-02-01 |

**Ticket Number Prefixes:**
- CVR = Custom Video Request
- VCS = Video Call Scheduling
- CRQ = Content Request
- INQ = General Inquiry
- URG = Urgent Alert

## What You DON'T Need to Migrate

These are auto-generated by the system:
- **TicketHistory** - Created automatically when tickets change
- **TicketComment** - Added through the app
- **Notification** - Generated automatically
- **AuditLog** - Recorded automatically
- **SystemSetting** - Configured in app settings
